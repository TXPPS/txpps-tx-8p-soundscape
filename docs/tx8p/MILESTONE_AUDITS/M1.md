# Milestone 1 — v3 Refinement Audit

Status: **Awaiting visual approval** — audio not yet started.

## 1. Change summary (v2 → v3)

v2 established the JX-8P-inspired visual identity (gray chassis, charcoal
programming strip, cream program buttons, smoked amber LCD, wide ivory
keyboard). v3 keeps that visual system verbatim and refactors the
**information architecture** to match the TX27 workflow:

- All synthesis sections are no longer visible at once.
- A **primary tab row** (OSC / FILTER / ENV / LFO / MOD / FX / VOICE)
  sits directly below the identity bar; only one editor is active at a
  time.
- Each editor uses **sub-tabs** (e.g. OSC → OSC 1 / OSC 2 / SUB-NOISE /
  MIX) plus a focused parameter row and single value slider.
- The large red PANIC has been removed from the performance panel and
  replaced by a **compact "!" hardware button** in the top-right corner
  of the identity bar, paired with a **compact Settings gear button**.
- The performance strip now contains only Bend / Mod levers, Octave,
  Hold, and a read-only mode indicator; detailed voice-mode editing
  lives on the VOICE tab.

## 5. Panic + Settings pair

- Location: far-right of the identity bar, paired as `[Settings] [!]`.
- Visible footprint: 32×32 hardware button; touch target ≥ 44×44 via
  `minWidth` / `minHeight`.
- Panic: muted red gradient, cream `AlertOctagon` glyph, accessible
  label + `title` "Panic — Stop All Notes". Flashes the LCD with
  `PANIC / ALL NOTES OFF` on activation. Note-off wiring lands in M2.
- Settings: cream hardware button, `Settings` gear glyph, opens a
  chassis-styled Settings dialog placeholder.

## 6. Persistent shell

Switching tabs / sub-tabs does **not** move or resize:

- identity mark & product plate
- preset LCD
- Prev / Next / Library
- Settings + Panic
- tab row itself
- performance strip
- keyboard

Selected tab and sub-tab persist through preset changes and viewport
resizes (Zustand-backed).

## 7. Responsive behavior

- **Desktop (≥ lg)**: two-column control surface — perf column on
  the left, focused editor on the right, wide keyboard docked at
  the bottom.
- **Tablet / md**: single-row identity bar, tabs full-width, editor
  above the perf column, keyboard docked.
- **Phone portrait**: two-row identity (row 1: brand + Settings + `!`;
  row 2: LCD + Prev/Next/Library). Tabs horizontally scroll if needed.
  Editor above perf strip; keyboard docked.
- **Phone landscape**: identity bar compresses, tab row remains,
  editor + perf + keyboard stack.

Motion respects `prefers-reduced-motion`. Reduced-motion path
neutralises transition-duration in `src/styles.css`.

## 8. LCD behavior

- Tab change: flashes `SECTION / <LONG LABEL>` for 600 ms, then
  returns to preset view.
- Parameter select / edit: flashes `<SUB>·<PARAM>` with a bar-graph
  + percentage for 900 ms, then returns.
- Panic: flashes `PANIC / ALL NOTES OFF` for 700 ms.
- Preset display persists as the resting state.

## 9. Visual continuity

Preserved from v2: medium warm gray chassis, charcoal programming
surfaces, cream/red/amber/blue button family, smoked amber LCD, thin
technical labels, restrained hardware shadows, wide ivory keyboard.
No neon accents, no oversized rotary knobs, no plugin-card styling,
no pill navigation. Tabs render as physical mode buttons with a top
LED dot in the section accent color when active.
