# Milestone 1 — Revised Audit (v2)

Status: **Awaiting visual approval** — audio not yet started.

## 1. Root cause of the original visual mismatch

The first pass of Milestone 1 was assembled from generic "modern soft-synth"
primitives (deep black cards, glowing neon-green LCD, oversized rotary
knobs, vertically stacked panel dashboard) rather than from the supplied
Roland JX-8P reference. It read as a contemporary VST plugin, not as a
programmable 1980s Japanese hardware polyphonic synthesizer, so it failed
the flagship TXPPS visual identity brief.

## 4. Layout — old vs new

**v1 (rejected):** Vertically stacked mobile-plugin layout — top LCD panel,
horizontal panel-tab row, side-by-side perf+panels grid, small keyboard at
bottom. Deep rounded cards. Whole surface read as three floating black
boxes.

**v2 (this revision):** Wide desktop-first horizontal chassis:

1. **Identity bar** (light gray, top, full width) — TXPPS mark, `TX-8P` in
   large charcoal with a small red `HYBRID` secondary mark, `HYBRID POLY
   SYNTHESIZER · 8 VOICES` subtitle, small POWER LED, compact smoked LCD,
   Write/Compare/Edit program buttons.
2. **Chassis seam** — thin bevel between upper gray and lower gray zones.
3. **Control surface** — two columns:
   - **Left performance column** (~190 px) with lever sliders (Bend / Mod),
     octave stepper with mini LCD readout, cream Hold, cream Poly/Mono/Uni
     voice buttons, and a **separated red Panic** at the bottom of the
     column so it can't be hit by accident.
   - **Right programming strip** — long charcoal panel with silkscreened
     section headers (Patch / Oscillator / Filter / Envelope / Modulation /
     Effects) and rows of small rectangular program buttons in cream, muted
     red, muted amber, and muted blue. Selecting a button lights its LED
     dot, names the parameter in the strip, and exposes a compact value
     slider for editing.
4. **Wide keyboard** — 3 octaves of restrained ivory whites and dark
   charcoal blacks recessed into the chassis, spanning the full width.

## 5. Control-system redesign

Removed the four-oversized-knobs-per-section presentation. The primary
surface is now **button-driven programming**: select a group, select a
parameter, adjust with the single edit slider and read the value on the
LCD. Rotary knobs are reserved for the future detailed-panel view (M4+)
and never dominate the chassis.

Performance controls are hardware-style: narrow vertical **lever
sliders** for bend and mod (bend is spring-return), rectangular cream
program buttons for hold/octave/voice, and Panic isolated in its own red
zone at the bottom.

## 6. Display redesign

The LCD is now a **compact smoked-glass window** (~22ch × 2 lines) sitting
inside the light-gray identity bar. Smoked dark background, subdued amber
characters, thin scanlines, subtle glass sheen, minimal glow. Default line
1 shows `Fxxx  <Preset Name>`, line 2 shows the voice mode and octave;
editing a parameter temporarily shows a labelled bar-graph value. No large
neon screen anywhere on the chassis. The full library browser (M6) can
still use a richer green-screen identity as a modal.
